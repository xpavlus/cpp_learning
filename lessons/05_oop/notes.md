# Урок 05 — ООП (Классы и объекты)

## Классы в C++

В Python ты уже знаком с `class`. В C++ концепция та же, но есть ключевые отличия:

- **Модификаторы доступа**: `public`, `private`, `protected` — определяют, кто может обращаться к членам класса.
- По умолчанию все члены класса — **`private`** (в отличие от Python, где всё публично).
- **Конструктор** — специальный метод с тем же именем, что и класс. Вызывается при создании объекта.
- **Деструктор** — `~ClassName()` — вызывается при уничтожении объекта (аналог `__del__` в Python, но надёжный).

```cpp
class Rectangle {
private:
    double width;
    double height;

public:
    // Конструктор
    Rectangle(double w, double h) : width(w), height(h) {}

    // Метод
    double area() const {
        return width * height;
    }
};

int main() {
    Rectangle r(3.0, 4.0);
    std::cout << r.area(); // 12
}
```

### Ключевые отличия от Python

| Python | C++ |
|--------|-----|
| `def __init__(self, x)` | `MyClass(int x) : field(x) {}` |
| `self.x = x` | поле объявляется в классе отдельно |
| всё публично по умолчанию | всё приватно по умолчанию в `class` |
| `__del__` ненадёжен | деструктор вызывается гарантированно |
| нет статической типизации полей | поля имеют явный тип |

### Список инициализации конструктора

```cpp
// Вместо:
MyClass(int x) { this->x = x; }

// Предпочтительный способ:
MyClass(int x) : x(x) {}
```

Список инициализации (`: field(value)`) — более эффективный способ задавать начальные значения полей.

### `const` у методов

Метод, помеченный `const`, обещает не изменять объект:
```cpp
double area() const { ... } // не меняет width/height
```
